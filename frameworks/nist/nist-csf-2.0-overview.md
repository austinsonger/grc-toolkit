# NIST Cybersecurity Framework (CSF) 2.0

**Framework**: NIST Cybersecurity Framework
**Version**: 2.0
**Published**: February 26, 2024
**Authority**: National Institute of Standards and Technology (NIST)
**Applicable To**: All organizations (voluntary framework)

---

## Overview

The NIST CSF 2.0 provides guidance for organizations to manage and reduce cybersecurity risk. Version 2.0 expands the original framework with a new "Govern" function and improved guidance for implementation.

## Core Functions

### GV – GOVERN (New in 2.0)
Establishes and monitors organizational cybersecurity risk management strategy, expectations, and policy.

| Category | Category ID | Description |
|---|---|---|
| Organizational Context | GV.OC | Circumstances surrounding the organization's cybersecurity risk management decisions |
| Risk Management Strategy | GV.RM | Organizational priorities, constraints, risk tolerance, and assumptions |
| Roles & Responsibilities | GV.RR | Cybersecurity roles, responsibilities, and authorities |
| Policy | GV.PO | Organizational cybersecurity policy established and communicated |
| Oversight | GV.OV | Results of organization-wide cybersecurity risk management reviewed |
| Cybersecurity Supply Chain | GV.SC | Cyber supply chain risk management activities |

### ID – IDENTIFY
Develop organizational understanding to manage cybersecurity risk to systems, people, assets, data, and capabilities.

| Category | Category ID | Description |
|---|---|---|
| Asset Management | ID.AM | Assets are inventoried and managed |
| Risk Assessment | ID.RA | Risk to operations and assets is identified and prioritized |
| Improvement | ID.IM | Improvements to organizational cybersecurity risk management |

### PR – PROTECT
Develop and implement safeguards to ensure delivery of critical services.

| Category | Category ID | Description |
|---|---|---|
| Identity Management & Access Control | PR.AA | Access to assets is managed |
| Awareness and Training | PR.AT | Personnel trained on cybersecurity responsibilities |
| Data Security | PR.DS | Data managed consistent with risk strategy |
| Platform Security | PR.PS | Hardware and software managed |
| Technology Infrastructure Resilience | PR.IR | Security architectures manage resilience |

### DE – DETECT
Develop and implement activities to identify the occurrence of a cybersecurity event.

| Category | Category ID | Description |
|---|---|---|
| Continuous Monitoring | DE.CM | Assets monitored to find anomalies |
| Adverse Event Analysis | DE.AE | Anomalies analyzed to characterize incidents |

### RS – RESPOND
Develop and implement activities for detected cybersecurity incidents.

| Category | Category ID | Description |
|---|---|---|
| Incident Management | RS.MA | Incident response activities managed |
| Incident Analysis | RS.AN | Investigations conducted for effective response |
| Incident Response Reporting | RS.CO | Response activities coordinated with stakeholders |
| Incident Mitigation | RS.MI | Activities to prevent expansion of events |

### RC – RECOVER
Develop and implement activities to maintain resilience and restore capabilities.

| Category | Category ID | Description |
|---|---|---|
| Incident Recovery Plan Execution | RC.RP | Restoration activities executed |
| Incident Recovery Communication | RC.CO | Restoration activities coordinated with stakeholders |

## Implementation Tiers

| Tier | Name | Description |
|---|---|---|
| 1 | Partial | Ad hoc, reactive practices |
| 2 | Risk Informed | Risk management practices approved but not organization-wide policy |
| 3 | Repeatable | Formally approved risk management practices |
| 4 | Adaptive | Cybersecurity practices adapted based on lessons learned |
